Estructuras de Control en JavaScript

Secuencia y Sintaxis

El orden en el que se ejecutan las instrucciones de un programa, es por defecto, secuencial con lectura lineal con preferencia la primera que aparece y luego la siguiente instrucción.

  • Un programa se escribirá como una sucesión de instrucciones o sentencias.
  • Al final de cada sentencia se utilizará un punto y coma para indicar el final de la instrucción.
  • Pueden agruparse una serie de sentencias en un bloque encerrándolas entre llaves.

Estructuras de Control

A veces es necesario alterar el orden de las sentencias, para ello se utilizan las estructuras de control.

  • Estructuras de decisión.: IF, IF...ELSE, SWITCH
  • Estructuras de repetición.
  • Estructuras de selección.

Estructuras de decisión

En JavaScript se pueden tomar decisiones utilizando tres tipos de estructuras:

  1. If
  2. If else
  3. Switch

En las estructuras de decisión hay dos expresiones de ruptura:

  1. break: La instrucción break pone fin al bloque y hace que el programa salte a la instrucción siguiente a la sentencia.
  2. default: La instrucción default es opcional, su finalidad es que se ejecute algún código cuando ninguna de las condiciones se cumple.

Una sentencia condicional es una instrucción en la que se hace una comparación y según el resultado verdadero o falso (true or false) de la misma, el programa seguirá ejecutando una u otra instrucción.

La estructura de decisión más simple que se puede tomar es la condicional.

  • If:

    Es un condicional que sirve para realizar unas u otras operaciones en función de una expresión.

    Una sentencia condicional ejecuta u omite una serie de sentencias dependiendo de si la comprobación da verdadero o falso.

    La sintaxis es:

    
if (expresión que debe cumplirse) {
   acción que se realizará
  }

    Solo si verdaderamente se cumple la expresión) entonces la acción se realizará. 

    
if (“llueve”) { 
  alert(“me quedo en casa”);
}

    Nota: Las llaves que incluyen la acción son opcionales, y lo mismo el sangrado. Si encontráis código que no los tenga, no quiere decir que sea incorrecto, pero es muy recomendable ponerlos.
